Representación y visualización de datos¶

El aprendizaje automtico trata de ajustar modelos a los datos; por esta razn, empezaremos discutiendo como los datos pueden ser representados para ser accesibles por el ordenador. Adems de esto, nos basaremos en los ejemplos de matplotlib de la seccin anterior para usarlos para representar datos.

Datos en scikit-learn¶

Los datos en scikit-learn, salvo algunas excepciones, suelen estar almacenados en arrays de 2 dimensiones, con forma [n_samples, n_features]. Muchos algoritmos aceptan tambin matrices scipy.sparse con la misma forma.

  • n_samples: este es el nmero de ejemplos. Cada ejemplo es un item a procesar (por ejemplo, clasificar). Un ejemplo puede ser un documento, una imagen, un sonido, un vdeo, un objeto astronmico, una fila de una base de datos o de un fichero CSV, o cualquier cosa que se pueda describir usando un conjunto prefijado de trazas cuantitativas.
  • n_features: este es el nmero de caractersticas descriptoras que se utilizan para describir cada item de forma cuantitativa. Las caractersticas son, generalmente, valores reales, aunque pueden ser categricas o valores discretos.

El nmero de caractersticas debe ser fijado de antemano. Sin embargo, puede ser extremadamente alto (por ejemplo, millones de caractersticas), siendo cero en la mayora de casos. En este tipo de datos, es buena idea usar matrices scipy.sparse que manejan mucho mejor la memoria.

Como ya comentamos en la seccin anterior, representamos los ejemplos (puntos o instancias) como filas en el array de datos y almacenamos las caractersticas correspondientes, las "dimensiones", como columnas.

Matrices scatterplot¶

En lugar de realizar los plots por separado, una herramienta comn que utilizan los analistas son las matrices scatterplot.

Estas matrices muestran los scatter plots entre todas las caractersticas del dataset, as como los histogramas para ver la distribucin de cada caracterstica.

In [11]:
import pandas as pd
    
iris_df = pd.DataFrame(iris.data, columns=iris.feature_names)
pd.plotting.scatter_matrix(iris_df, c=iris.target, figsize=(8, 8));

Otros datasets disponibles¶

Scikit-learn pone a disposicin de la comunidad una gran cantidad de datasets. Vienen en tres modos:

  • Packaged Data: pequeos datasets ya disponibles en la distribucin de scikit-learn, a los que se puede acceder mediante sklearn.datasets.load_*
  • Downloadable Data: estos datasets son ms grandes y pueden descargarse mediante herramientas que scikit-learn ya incluye. Estas herramientas estn en sklearn.datasets.fetch_*
  • Generated Data: estos datasets se generan mediante modelos basados en semillas aleatorias (datasets sintticos). Estn disponibles en sklearn.datasets.make_*

Puedes explorar las herramientas de datasets de scikit-learn usando la funcionalidad de autocompletado que tiene IPython. Tras importar el paquete datasets de sklearn, teclea

o

o

para ver una lista de las funciones disponibles

Aqu la etiqueta es directamente el dgito que representa cada ejemplo. Los datos consisten en un array de longitud 64... pero, qu significan estos datos?

Una pista viene dada por el hecho de que tenemos dos versiones de los datos: data y images. Vamos a echar un vistazo a ambas:

Vamos a visualizar los datos. Es un poco ms complejo que el scatter plot que hicimos anteriormente.

In [19]:
# Configurar la figura
fig = plt.figure(figsize=(6, 6))  # tamao en pulgadas
fig.subplots_adjust(left=0, right=1, bottom=0, top=1, hspace=0.05, wspace=0.05)

# mostrar algunos dgitos: cada imagen es de 8x8
for i in range(64):
    ax = fig.add_subplot(8, 8, i + 1, xticks=[], yticks=[])
    ax.imshow(digits.images[i], cmap=plt.cm.binary, interpolation='nearest')
    
    # Etiquetar la imagen con el valor objetivo
    ax.text(0, 7, str(digits.target[i]))

Ahora podemos saber que significan las caractersticas. Cada caracterstica es una cantidad real que representa la oscuridad de un pxel en una imagen 8x8 de un dgito manuscrito.

Aunque cada ejemplo tiene datos que son inherentemente de dos dimensiones, la matriz de datos incluye estos datos 2D en un solo vector, contenido en cada fila de la misma.
